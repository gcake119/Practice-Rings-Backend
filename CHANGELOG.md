# Changelog

All notable changes to this project will be documented in this file.

## [1.2.0] - 2025-12-12

### Added
- 單日記錄卡片（Day View）規格：
  - 定義點擊最近紀錄圈圈開啟單日卡片的流程與資料欄位（圈圈完成度、各模式分鐘數 / 目標分鐘數、當日筆記）。
  - 新增「前一天 / 後一天」導覽行為說明與可切換的日期範圍限制。
- 月檢視卡片（Month View）規格：
  - 描述整月行事曆格子的排列方式（週一作為一週起點）與三合一小同心圓的顯示邏輯。
  - 定義從單日卡片切換至月檢視、以及點擊日期格子返回單日卡片的互動流程。
- 手機滑動手勢：
  - 新增在行動裝置上透過上下滑動月份卡片切換「上個月 / 下個月」的需求說明。
  - 指定僅在窄螢幕裝置啟用手勢，桌機仍以按鈕操作為主。

### Changed
- 更新歷史紀錄區段說明：
  - 明確區分「最近七天橫向小圈圈視圖」與「月檢視行事曆」兩種模式及切換方式。
  - 補充日檢視與月檢視的 UI 導覽按鈕行為（Day View 使用前後一天、Month View 使用上月下月）。
- 前端狀態設計描述調整：
  - 補充 `currentViewDate` 與 `currentCalendarDate` 的定義與在日/月檢視切換時的使用規則。

## [1.1.0] - 2025-12-04

### Added
- 補充 Phase 2 規格：描述前端 Doraemon 主題 UI、toast 通知與粉圓體字型需求。
- 新增「手動輸入時間」流程說明，包括時間格式、驗證規則與錯誤處理策略。
- 定義「三項目總時數以 6 小時為 100%」的計算方式，作為前端進度條與勵志文案的依據。
- 描述歷史紀錄卡片的週視圖與月視圖需求（最近七天卡片、月曆 Grid、週一為一週起點）。

### Changed
- 調整前後端介面文件，將原本零散的需求整理為 Phase 2 單一規格文件。
- 更新登入流程說明：
  - 保留後端以 token 驗證受保護路由的機制，但前端不再將 token 存入 localStorage，自動登入行為暫不實作，使用者重新整理頁面後需再次登入。
